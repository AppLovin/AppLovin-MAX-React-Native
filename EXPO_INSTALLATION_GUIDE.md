# Expo Installation Guide for react-native-applovin-max

**Note:** This package cannot be used in the "Expo Go" app because it requires [custom native code](https://docs.expo.dev/workflow/customizing/).


## Step-by-step Instruction

1. Install `react-native-applovin-max` in your Expo root.

```
% npm i react-native-applovin-max
```

You should now have the directory `node_modules/react-native-applovin-max`.

2. Copy the files [app.plugin.js](https://github.com/AppLovin/AppLovin-MAX-React-Native/blob/feat/add_expo_plugins/app.plugin.js) and [plugin](https://github.com/AppLovin/AppLovin-MAX-React-Native/tree/feat/add_expo_plugins/plugin) directory from the [feat/add_expo_plugins](https://github.com/AppLovin/AppLovin-MAX-React-Native/tree/feat/add_expo_plugins) branch of the [AppLovin-MAX-React-Native](https://github.com/AppLovin/AppLovin-MAX-React-Native) repository.

```
% cp app.plugin.js node_modules/react-native-applovin-max
% cp -r plugin node_modules/react-native-applovin-max
```

Note: The `react-native-applovin-max` package may include these files in the future release.

3. Configure your `app.json` file as instructed in the section below.

4. Run `expo prebuild`.

```
% npx expo prebuild --clean
```

5. Run the build for Android and iOS.

```
% npx expo run:android
% npx expo run:ios
```

## Plugin Configuration

Add the [config plugin](https://docs.expo.dev/config-plugins/introduction/) to the plugins array of your `app.json`:

```json
{
  "expo": {
    ...
    "plugins": [
      [
        "react-native-applovin-max"
      ]
    ]
  }
}
```

## Configurable Items
Here is a list of the items you can configure:

Android:
- Gradle Dependencies
- AndroidManifest Meta Data

iOS:
- CocoaPods Dependencies
- Info.plist Keys and Values
- SKAdNetwork Identifiers

Other than SKAdNetwork Identifiers, the items are self-explanatory. Here is an example of the plugin configuration:

```json
"plugins": [
  [
    "react-native-applovin-max",
    {
      "android": {
        "dependencies": [
          {
            "dependency": "com.applovin.mediation:google-adapter:+"
          },
          {
            "repository": "https://android-sdk.is.com",
            "dependency": "com.applovin.mediation:ironsource-adapter:+"
          }
        ],
        "metaDataManifest": [
          {
            "name": "com.google.android.gms.ads.APPLICATION_ID",
            "value": "your_admob_app",
          }
        ]
      },
      "ios": {
        "dependencies": [
          "AppLovinMediationGoogleAdapter",
          "AppLovinMediationIronSourceAdapter"
        ],
        "skAdNetworkIdFile": "skadnetwork_ids.xml",
        "infoPlist": [
          {
            "key": "GADApplicationIdentifier",
            "value": "your_admob_app_id"
          }
        ]
      }
    }
  ]
]
```

## SKAdNetwork Identifiers

You can specify a file that contains a list of SKAdNetwork identifiers generated by [Info.plist Generator](https://developers.applovin.com/en/ios/overview/skadnetwork#infoplist-generator).  Select the mediated ad networks that you integrated with MAX in the Info.plist Generator, open the tabe below the Info.plist Generator, and copy and paste the identifies to a file.

```
$ cat skadnetwork_ids.xml
<dict>
   <key>SKAdNetworkIdentifier</key>
   <string>22mmun2rn5.skadnetwork</string>
</dict><dict>
   <key>SKAdNetworkIdentifier</key>
   <string>238da6jt44.skadnetwork</string>
</dict><dict>
   <key>SKAdNetworkIdentifier</key>
   <string>24t9a8vw3c.skadnetwork</string>
</dict><dict>
...
```
